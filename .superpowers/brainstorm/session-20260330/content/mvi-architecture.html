Approach 1: Core-Out Architecture Migration

Transitioning from MVVM to strict MVI for core screens (Home & Player)

Current: MVVM + StateFlow
View (Compose)
↓ ↑
ViewModel
private val _uiState = MutableStateFlow(...)
val uiState = _uiState.asStateFlow()

fun onAction(...) {
  // Manual state updates
  _uiState.value = ...
}
Proposed: Orbit MVI (The "Clean Break")
View (Compose)
↓ (Intents)
MVI Store / Container
↑ (State / SideEffects)
intent {
  reduce {
    state.copy(loading = true)
  }
  // Async logic
  postSideEffect(Toast("..."))
}

Migration Roadmap (A + C)

0

Phase 0: Documentation Alignment

Align CURRENT_STATUS.md, KNOWN_ISSUES.md, and ROADMAP.md. Establish a "single source of truth."

1

Phase 1: Architecture Core

Implement :core:architecture with Orbit MVI. Abstract common state logic for the "Clean Break."

2

Phase 2: High-Impact Migration (Home/Player)

Migrate the most complex screens to MVI. This also addresses Phase 3 of your existing plan (Refactoring Oversized Composables).

3

Phase 3: Dependency Stabilization

Stabilize Media3, Navigation, and Lifecycle dependencies (Phase 1 of your existing plan).

Pros

  • Solves state-management complexity for Home/Player early.
  • Establishes a repeatable modern standard for the whole app.
  • Directly reduces the "large composable" maintenance burden.

Cons

  • Higher initial learning curve for the new pattern.
  • Requires temporary split architecture (MVVM vs MVI) until full migration.
